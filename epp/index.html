The extensible Java pre-processor EPP

Yuuji ICHISUGI
National Institute of Advanced Industrial Science and Technology(AIST)

Sorry, I have stopped development of EPP.

EPP is a powerful framework which supports implementation of the following tools:
  • Pre-processors which extend Java language
  • Java source-code analyzers (e.g. metrics tool, cross reference, ...)
  • Java source-code translators (e.g. source-level optimizer, obfuscator, ...)
EPP has extensible parser and type-checker. Its architecture is powerful but simple and easy to understand, especially, if you have basic knowledge of lisp and compilers.

1999-07-01 (EPP 1.1.0 beta 7)

  • This version uses JavaClass API (developed by Markus Dahm) to analyze class files during type-checking pass.
  • If you use jview, you should add classes.zip to your CLASSPATH explicitly. If it is not included, EPP can not find class files of Java libraries during type-checking pass.

  • -class-info and -file-signature options are added.

JavaClass

EPP internally uses the package "JavaClass" . JavaClass is a LGPL class library for byte-code engineering, developed by Markus Dahm. To obtain full-set of class files, source files and documents of JavaClass, please visit the following URLs.

http://www.inf.fu-berlin.de/~dahm/JavaClass/index.html
or
http://staff.aist.go.jp/y-ichisugi/epp/JavaClass/JavaClass3.3.2.tar.gz

Patch for JavaClass3.3.2 which passes J2SE 1.4 classfiles:
http://staff.aist.go.jp/y-ichisugi/epp/JavaClass/JavaClass/jdk14.patch
